MVC

发表于2025-04-23|更新于2025-04-23
|浏览量:

MVC（Model-View-Controller） 是一种经典的软件架构模式，用于将应用程序的逻辑、数据和用户界面分离，提升代码的可维护性和扩展性。最初由 Trygve Reenskaug 在 1978 年提出，现广泛应用于 Web 开发（如 Ruby on Rails、Django）、桌面应用（如 Java Swing）和移动开发（如 iOS）中。

MVC 的三大组件

组件 职责 示例（Web 应用场景）
Model（模型） 处理数据和业务逻辑（如数据库操作、计算） 从数据库查询用户信息，验证密码是否正确
View（视图） 展示数据（用户界面），不包含业务逻辑 渲染 HTML 页面，显示用户个人资料
Controller（控制器） 接收用户输入，协调 Model 和 View（中介角色） 处理表单提交，调用 Model 验证数据并更新 View

工作流程（以用户登录为例）

  1. 用户操作：在登录页面输入账号密码，点击提交。
  2. Controller 接收请求：
    • 解析表单数据（如 username=admin&password=123）。
  3. Model 处理逻辑：
    • 查询数据库验证账号密码是否匹配。
  4. View 更新界面：
    • 若成功：跳转到首页。
    • 若失败：显示错误提示。

MVC 的优势

  1. 分离关注点
    • 开发者可以独立修改界面（View）或业务逻辑（Model）而不互相影响。
  2. 代码复用
    • 同一个 Model 可被多个 View 复用（如网页版和移动版共享数据逻辑）。
  3. 易于协作
    • 设计师负责 View，工程师专注 Model 和 Controller。

MVC 的变体与演进

  1. MVVM（Model-View-ViewModel）
    • 用于前端框架（如 Vue、Angular），通过数据绑定自动同步 View 和 Model。
  2. MVP（Model-View-Presenter）
    • Controller 改为 Presenter，View 与 Model 完全解耦（常见于 Android 开发）。

常见误区

  • View 不能直接访问 Model：必须通过 Controller 协调。
  • Controller 不应包含业务逻辑：复杂逻辑应放在 Model 中。
  • MVC ≠ 三层架构：MVC 是表现层模式，三层架构（表现层/业务层/数据层）是更宏观的分层。

现代 Web 中的 MVC

  • 后端 MVC：Rails（Ruby）、Django（Python）等框架。
  • 前端 MVC：Angular、React（更接近 MVVM）。
  • SPA 中的 MVC：前端通过 API 与后端 Model 交互，前端 Router 充当 Controller。

MVC 通过清晰的职责划分，帮助开发者构建可维护的大型应用，是理解现代框架设计的基础。
